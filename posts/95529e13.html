Node.js——fs模块

发表于2023-05-07|Node.js学习记录
|字数总计:1.8k|阅读时长:7分钟|阅读量:

Node.js的fs模块能够与文件系统进行交互。

fs模块介绍

fs（File System）是Node.js的内置模块，可以实现与硬盘的交互，例如文件的创建、删除、重命名、移动，以及文件的写入与读取。

fs模块中的方法有同步与异步之分。

下面我将介绍fs模块的基本操作：

文件写入

将数据保存到文件中，我们平常下载文件、安装软件的时候，就是将文件写入了磁盘中。

1. writeFile异步写入

语法：fs.writeFile(file, data[, options], callback)

说明：

参数 含义
file 指定文件路径
data 待写入的数据
options 配置选项
callback 回调函数

使用：

2.writeFileSync同步写入

语法：fs.writeFile(file, data[, options])

说明：参数部分与异步写入一致。

使用：

3. appendFile/appendFileSync追加写入

同样分为同步追加写入和异步追加写入。

语法：

fs.appendFile(file, data[, options], callback)

fs.appendFileSync(file, data[, options])

说明：参数部分与上述方法基本一致。

使用：

4. creatWriteStream流式写入

语法： fs.createWriteStream(path[, options])

说明：参数含义仍然一致。

使用：

对比：流式写入方式适用于 大文件写入或者频繁写入 的场景, writeFile 适合于 写入频率较低的场景

文件读取

从文件中读取数据，平时电脑开机、程序运行、播放视频、查看图片都是在读取文件。

1. readFile异步读取

语法： fs.readFile(path[, options], callback)

说明：

参数 含义
file 指定文件路径
options 配置对象
callback 回调函数

使用：

2.readFileSync同步读取

语法： fs.readFileSync(path[, options])

说明：参数与上述方法基本一致。

使用：

3. creatReadStream流式读取

语法： fs.createReadStream(path[, options])

说明：参数与上述方法基本一致。

使用：

文件重命名与移动

rename/renameSync

语法：

fs.rename(oldPath, newPath, callback)

fs.renameSync(oldPath, newPath)

说明：

参数 含义
oldPath 当前路径
newPath 新的路径
callback 回调函数

使用：

文件删除

将文件从当前目录移除。

1.unlink/unlinkSync

语法：

fs.unlink(path, callback)

fs.unlinkSync(path)

说明：

参数 含义
path 文件路径
callback 回调函数

使用：

2.rm

语法：fs.rm(path, callback)

参数：与上述方法一致。

使用：

操作文件夹

1. mkdir/mkdirSync创建文件夹

语法：

fs.mkdir(path[, options], callback)

fs.mkdirSync(path[, options])

说明：

参数 含义
path 文件夹路径
options 配置对象
callbak 回调函数

使用：

2. readdir/readdirSync读取文件夹

语法：

fs.readdir(path[, options], callback)

fs.readdirSync(path[, options])

说明：参数与上述方法一致。

使用：

3. rm/rmdir删除文件夹

语法：

fs.rm(path[, options], callback)

fs.rmdir(path[, options], callback)

说明：参数与上述方法一致。

使用：

注：rmdir也能递归删除文件夹，但是recursive配置已经废弃了，所以不推荐使用。

stat/statSync查看资源状态

语法：

fs.stat(path[, options], callback)

fs.statSync(path[, options])

说明：参数含义同上。

使用：

文件路径

上述方法的第一个参数的都是指文件的路径，而文件的路径有两种写法：

  • 相对路径：./a.txt（当前目录下）、a.text（当前目录下）、../a.txt（上一级目录下）
  • 绝对路径：C:/a.txt、/a.txt

说明：相对路径是指相对命令行的工作目录，而非文件所在的目录。所以在不同目录下执行文件，会出现一些问题。要想解决这个问题，要么使用绝对路径，要么使用__dirname（这个变量始终保存当前文件所在目录的绝对路径）拼接。

当然，与path模块搭配使用，可以生成更加正确的路径。而且也推荐与path搭配使用。
